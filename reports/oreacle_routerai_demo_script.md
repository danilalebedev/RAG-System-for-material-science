# Oreacle RouterAI demo script

Актуальный сценарий для локального показа организаторам после сборки `routerai_bge_m3`.

## Что показываем

Главная идея: Oreacle не просто ищет документы. Он собирает инженерный ответ по внутреннему корпусу, показывает
доказательства, сравнивает локальные материалы с внешней литературой и выгружает проверяемый отчет.

Демо URL:

```text
http://127.0.0.1:8501/
```

Рекомендуемый профиль в GUI:

```text
routerai_bge_m3
```

## Основной запрос

```text
Какие никелевые сплавы применяются в судостроении и какие режимы термообработки влияют на твердость?
```

Почему этот запрос удобен:

- есть материал, отрасль, процесс и свойство;
- хорошо демонстрирует decomposition/rewrite;
- дает смысл для local-vs-web comparison;
- подходит для режима методик и для web literature search.

## 90-секундный сценарий

1. Открыть GUI и выбрать `Поиск методик`.
2. Проверить, что RAG profile установлен в `routerai_bge_m3`.
3. Включить web-search, но Deep Search оставить выключенным для первого быстрого ответа.
4. Запустить основной запрос.
5. Во вкладке `Ответ` показать:
   - итоговый ответ;
   - confidence score;
   - ключевые слова и переформулировки запроса.
6. Во вкладке `Evidence` показать, что ответ собирается из нескольких потоков:
   - `Raw RAG`;
   - `Summary RAG`;
   - `Tables`;
   - fallback/status rows, если источник недоступен.
7. Во вкладке `Источники` показать web-search и локальные совпадения:
   - confidence `Высокая/Средняя/Низкая`;
   - причины релевантности: keyword/title/abstract/domain hits, квартиль, DOI/abstract.
8. Во вкладке `Графы` показать Knowledge Graph и local-vs-web method graph как быстрый визуальный слой.
9. Во вкладке `Графики` показать распределение публикаций по годам и базам данных.
10. Во вкладке `Отчеты` скачать:
    - RouterAI answer report;
    - обычный отчет ссылок;
    - полный PDF/DOCX;
    - `run_artifacts.zip`.

## Deep Search mini-flow

После первого быстрого ответа нажать Deep Search для уже найденных источников.

Показывать:

- summary по каждой статье;
- общий summary на русском языке;
- сравнение методик и диапазонов;
- отдельный Deep Search PDF/DOCX;
- обновленный ZIP с `web_document_summaries.jsonl`, `web_procedure_summaries.jsonl` и `comparison_report.json`.

## Что выделяет решение

- Единый embedding space: raw chunks + document summaries + procedure summaries через `baai/bge-m3`.
- Сценарный GUI: пользователь выбирает не технический pipeline, а тип задачи: литература, методики, свойства.
- Evidence-first ответ: можно открыть источники, локальные совпадения, графы и отчеты.
- Local vs world comparison: система не просто ищет статьи, а показывает что подтверждается, что есть только локально,
  что найдено во внешней литературе и где есть пробелы.
- Экспорт для проверки: организатор может скачать PDF/DOCX/ZIP и проверить ссылки, summaries, manifests и локальные
  файлы без копания в коде.
- Confidence по публикациям показывает не только score, но и причины релевантности, поэтому выдачу легче защищать перед
  экспертами.

## Что не обещать

- Не говорить, что web full text массово скачивается: сохраняются metadata, ссылки, excerpts и summaries.
- Не обещать production-grade библиометрию квартилей: текущий Q-score является demo/ranking heuristic.
- Не говорить, что graph полностью заменяет экспертную онтологию: текущий GUI показывает полезный слой навигации,
  который можно заменить/обогатить готовым graph backend.
