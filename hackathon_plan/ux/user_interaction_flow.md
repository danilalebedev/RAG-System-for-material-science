# Детализация взаимодействия с пользователем

## Цель интерфейса

Интерфейс должен выглядеть не как общий чат, а как рабочее место материаловеда: пользователь задает вопрос, система уточняет недостающие сущности, возвращает ответ с источниками, граф связей и список пробелов. Главный принцип: каждый вывод должен быть проверяемым через evidence.

## Основные режимы

| Режим | Когда нужен | Что показывает |
|---|---|---|
| Найти факты | "Что делали по сплаву X при режиме Y?" | краткий ответ, список экспериментов, источники |
| Сравнить эксперименты | "Как режимы Y1/Y2 влияли на свойство Z?" | таблицу сравнений, граф связей, противоречия |
| Найти пробелы | "Что не измеряли для материала X?" | отсутствующие свойства/режимы, близкие аналоги |
| Сформировать гипотезу | "Как улучшить KPI Z для сплава X?" | гипотезы с support/novelty/feasibility/risk |

## Экран 1: постановка вопроса

Элементы:

- строка запроса;
- переключатель режима: `Факты`, `Сравнение`, `Пробелы`, `Гипотезы`;
- фильтры: материал, свойство, режим, оборудование, лаборатория, год/период;
- переключатель `Только подтвержденные источниками ответы`;
- лимит ответа: кратко / подробно / таблица.

Пример запроса: "Что уже делали по никелевым сплавам при термообработке 900 C и как это влияло на твердость?"

## Экран 2: уточнение запроса

Если query parser не уверен, система не должна сразу отвечать. Она показывает короткие уточнения:

- "Под `никелевыми сплавами` найдены: Ni-Cr, Ni-Cu, Ni-Al, ЭП718. Что выбрать?"
- "Свойство `прочность` встречается как tensile strength, yield strength, ultimate strength. Какое нужно?"
- "Режим `отжиг` найден при 850-950 C и 1-4 ч. Использовать весь диапазон?"

Важно: уточнения строятся из справочников и графа, а не только из LLM. Это снижает риск галлюцинаций.

## Экран 3: ответ

Ответ состоит из пяти вкладок.

| Вкладка | Содержимое |
|---|---|
| Answer | 3-7 строк вывода: что найдено, какой эффект, насколько надежно |
| Evidence | источники, чанки, таблицы, страницы/URL, extracted triples |
| Graph | интерактивный граф material-process-property-experiment |
| Gaps | недостающие свойства, режимы без измерений, противоречивые claims |
| Benchmark | время ответа, top-k источники, faithfulness/self-check |

## Экран 4: действия после ответа

Пользователь должен уметь:

- открыть исходный документ на нужной странице/секции;
- добавить найденный эксперимент в сравнение;
- сузить материал/режим/свойство;
- попросить гипотезу на основе найденных пробелов;
- отметить ответ как полезный/неверный/неполный.

Оценка пользователя сохраняется в `feedback_log` и потом используется для ручной проверки retrieval.

## Диаграмма

Подготовлены две версии:

- `user_interaction_graph_readable.mmd` / `user_interaction_graph_readable.png` - читаемая картинка для отчета или слайда.
- `user_interaction_flow.mmd` / `user_interaction_flow.png` - более полный flow со всеми ветками.
