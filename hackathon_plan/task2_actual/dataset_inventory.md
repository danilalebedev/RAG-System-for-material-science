## Выводы для разработки

1. Ingestion должен быть PDF-first: основной объем и количество файлов — PDF.
2. Нужны парсеры DOCX/DOCM, PPTX, XLS/XLSX и архивов ZIP/RAR.
3. Большой объем PDF делает полную LLM-обработку всего корпуса слишком дорогой и медленной. LLM нужно использовать выборочно: extraction по релевантным секциям, procedure summaries, нормализация сущностей.
4. Для демо нужно заранее подготовить индексы; online ingestion на защите не должен быть критическим путем.
5. Источник/раздел из пути файла важен как метаданные: `Доклады`, `Журналы`, `Обзоры`, `Статьи`, `Материалы конференций`.
6. Вероятны отсканированные PDF и сложные таблицы, поэтому нужен fallback OCR/RouterAI PDF parser/ручной контроль качества.
