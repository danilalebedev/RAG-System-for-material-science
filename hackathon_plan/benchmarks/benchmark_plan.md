# Benchmark Plan

## 1. Retrieval

Использовать вопросы организаторов, примеры формирования гипотез и 20-50 вручную составленных вопросов по корпусу.

| Метрика | Что измеряет | Как считать |
|---|---|---|
| Recall@k | попал ли нужный источник в top-k | по размеченным source_id/chunk_id |
| nDCG@10 | насколько высоко ранжируются релевантные источники | graded relevance 0/1/2 |
| MRR | позиция первого правильного источника | 1/rank |
| context precision | доля полезного контекста в evidence pack | ручная/LLM-assisted проверка |
| graph hit rate | нашел ли граф нужные соседние сущности | по ожидаемым triples |

## 2. Extraction

| Метрика | Что измеряет |
|---|---|
| Entity F1 | материалы, свойства, режимы, оборудование, команды |
| Relation F1 | связи material-process-property-effect |
| JSON validity | доля валидных ответов extractor-а |
| Unit normalization accuracy | корректность температур, времени, давления, концентраций |
| Evidence-span coverage | у каждой сущности/связи есть ссылка на исходный фрагмент |
| Deduplication accuracy | одинаковые материалы/свойства не размножаются в графе |

## 3. Answer Quality

| Метрика | Что измеряет |
|---|---|
| Faithfulness | нет ли утверждений без поддержки в retrieved evidence |
| Answer relevance | отвечает ли текст на вопрос |
| Citation accuracy | каждая ключевая фраза привязана к корректному источнику |
| Completeness | перечислены ли все найденные эксперименты/свойства |
| Contradiction handling | система показывает конфликтующие выводы, а не усредняет их |
| Abstention quality | система честно говорит, когда данных нет |

Автоматические инструменты: RAGAS/ARES как быстрый слой, но финальный score лучше подтверждать ручной экспертной разметкой 20-30 вопросов.

## 4. Hypothesis Quality

| Метрика | Что измеряет |
|---|---|
| Support strength | сколько и насколько релевантных источников поддерживают гипотезу |
| Novelty | не повторяет ли гипотеза уже проведенный эксперимент |
| Feasibility | есть ли доступные материалы/оборудование/режимы |
| KPI impact | ожидаемое влияние на целевое свойство |
| Risk/uncertainty | есть ли противоречия и пробелы |
| Expert rank correlation | совпадает ли ранжирование с оценкой эксперта |

## 5. Скорость и ресурсы

Организаторы ценят экономию ресурсов, поэтому эти метрики нужно показывать отдельно.

| Метрика | Цель для демо |
|---|---|
| P50/P95 latency ответа | интерактивно, желательно до 10-30 секунд на готовом индексе |
| Время индексации демо-корпуса | уложиться в 1-3 минуты для демонстрационного набора |
| RAM/VRAM | явно указать для CPU/GPU режима |
| Размер LLM и embeddings | показать компактный локальный вариант |
| Стоимость запроса | 0 для локального режима или прозрачная оценка для разрешенного API |
