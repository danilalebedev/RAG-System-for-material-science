# Production Radar / Market Radar

## Summary

Production Radar is the Agent F compatibility name for the existing Oreacle Market Radar business intelligence layer.
Both names point to the same independent module under `app/market/*`.

The layer is intentionally separate from RAG, graph retrieval, and query orchestration internals. It prepares dashboard-ready production rows for business/product demos: commodity, producer or country, period, value, unit, source URL, confidence, source status, caveats, and chart payloads.

## CLI Alias

```powershell
.\.venv\Scripts\python.exe scripts\run_production_radar.py "Сколько никеля произвёл Норникель?"
```

`scripts/run_production_radar.py` is a thin wrapper around the same Market Radar logic used by:

```powershell
.\.venv\Scripts\python.exe scripts\run_market_radar.py "Сколько никеля произвёл Норникель?"
```

## Dashboard Row Schema

Production Radar exposes dashboard-ready rows in this compact schema:

```json
{
  "commodity": "...",
  "producer_or_country": "...",
  "period": "...",
  "value": "...",
  "unit": "...",
  "source_url": "...",
  "confidence": "high | medium | low"
}
```

The original normalized source rows are still available as `production_rows`; the compact rows are available as `dashboard_rows` in CLI output.

## Source Registry Stubs

The source registry includes metadata and live-parser placeholders for:

- Fedstat / Rosstat / EMISS
- World Steel Association
- International Aluminium Institute
- USGS Mineral Commodity Summaries
- Nornickel reports / interactive database
- RUSAL reports
- Severstal reports
- NLMK reports
- MMK reports
- Metalloinvest reports

Live public-source fetching/parsing is planned but intentionally not required for the stable hackathon demo. The current mode uses small demo-safe fallback fixtures and clearly marks source status as `fallback`.

## Demo-Safe Mode

No fragile scraping is performed. Numeric facts are not generated by LLM. If live loading is disabled or unavailable, the module returns small built-in fixtures, source status, and caveats so the GUI remains stable.
