# Final response contract

Use this at the end of a task.

## Code change response

Include:

1. Summary — concise description of what changed.
2. Files changed — list important files.
3. Validation — exact commands and results.
4. Tests not run — with reason.
5. Risks/follow-up — only if relevant.

Do not include excessive implementation detail unless the user asked for it.

## Investigation response

Include:

1. What was inspected.
2. Findings with evidence.
3. Recommended next step.
4. Uncertainties.

## Review response

Include:

1. Findings ordered by severity.
2. File/location for each finding.
3. Why it matters.
4. Suggested fix.
5. Overall risk and validation gaps.

## Language

Respond in the user's language unless the user asks otherwise.
